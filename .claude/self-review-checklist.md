# Claude Code 自查清单

每次完成功能改动后，必须按此清单逐项验证，**在说"做完了"之前走完**。

---

## 1. 持久化验证

**最容易漏**。改完任何会写入 IndexedDB/localStorage/后端 config 的逻辑后，必须：

- [ ] 操作完成后刷新页面 → 数据是否还在？
- [ ] 关闭浏览器标签页重新打开 → 状态是否恢复？
- [ ] 如果是异步写入（如 TTS 合成后写 audioUrl），检查：写入是在回调完成前还是完成后？

**注意**：单元测试的 fake IndexedDB 不模拟持久化。必须在真实浏览器里刷新验证。

---

## 2. 错误反馈验证

**第二容易漏**。每个 try/catch、每个 guard return 必须：

- [ ] 搜索全项目 `catch.*\{\s*\}` — 不能有空 catch 块
- [ ] 搜索 `catch.*\{.*\/\/` — 不能有静默注释（"silently ignore"）
- [ ] 每个 `if (condition) return` 之前 — 是否有 toast/warning 告知用户为什么操作没执行？
- [ ] API 调用失败时 — 用户看到的是"加载失败"还是空白页面？
- [ ] 后端挂了的时候 — 前端页面是否明确提示"后端不可用"而不是静默空白？

---

## 3. 加载/空/错误三态验证

每个数据驱动的组件/页面：

- [ ] **加载态**：数据未返回时显示什么？是 spinner 还是骨架屏还是空白？
- [ ] **空态**：数据为空数组时显示什么？有引导用户下一步操作的文案吗？
- [ ] **错误态**：数据获取失败时显示什么？有重试入口吗？

---

## 4. 边界值验证

- [ ] 字符串：空字符串、超长字符串、纯空格
- [ ] 数字：0、负数、超大值
- [ ] 数组：空数组、单元素、超大数组
- [ ] 文件：无效 JSON、错误格式、超大的文件
- [ ] 网络：后端不可用、超时、返回 500

---

## 5. 状态转换验证

- [ ] 快速双击按钮 — 是否触发了两次操作？
- [ ] 操作进行中点取消/停止 — 状态是否正确重置？
- [ ] 操作失败后 — 能否重试？按钮是否恢复可点击？
- [ ] 一个 async 操作未完成时切换页面 — 有没有内存泄漏/console error？

---

## 6. 浏览器验证

**不能只看代码和跑测试。** 必须打开浏览器实际走一遍：

- [ ] `npm run dev` + `cargo run` 启动全栈
- [ ] 实际执行核心用户路径（发消息、创建角色、合成语音...）
- [ ] 刷新页面，检查数据是否还在
- [ ] 打开 DevTools Console — 有无红色报错？
- [ ] 打开 DevTools Network — 有无重复 API 调用？有无失败的请求？

---

## 7. 回归检查

- [ ] `npx tsc --noEmit` 零错误
- [ ] `npx vitest run` 全部通过
- [ ] `cargo test` 全部通过
- [ ] `git diff --stat` 确认改动的文件都在预期内，没有意外修改
